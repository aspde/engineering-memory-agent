# 测试规则

## 测试框架

使用 pytest + pytest-asyncio（async 测试）。配置见 `pyproject.toml`。

## 测试原则

所有新增功能必须添加对应测试，保证已有测试通过，覆盖核心逻辑。

禁止：只修改代码不增加测试、删除已有测试绕过问题。

## Mock 原则

外部依赖必须 Mock：

- LLM API
- 第三方服务
- 外部网络请求

测试重点为业务逻辑、数据流转、状态变化。不依赖真实 LLM 响应进行测试。

## 分层测试要求

| 层级 | 测试重点 | 要求 |
|------|---------|------|
| Backend API | 正常请求、参数校验、异常处理、返回结果 | 使用测试客户端，不依赖真实外部服务 |
| Agent | 状态流转、节点执行、Tool 调用、Memory 检索 | 优先测试 workflow 和状态变化，Mock LLM |
| Memory | 数据保存、数据检索、相似度查询、上下文构建 | 使用测试数据，保持可重复 |
| Database | Schema、查询、迁移 | 使用测试数据库，禁止操作生产数据 |

## 提交前

运行 `pytest`，确保全部通过。其他提交前检查见 [git-workflow.md](git-workflow.md)。
