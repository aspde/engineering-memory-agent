# 测试规则

## 测试原则

所有新增功能必须考虑测试。

测试目标：

- 保证功能正确
- 防止已有功能回归
- 提高代码可维护性


---

# 测试框架

使用：

- pytest


异步测试：

- pytest-asyncio


要求：

- 支持 async 测试
- 保持测试执行稳定


---

# 测试目录规范

测试代码统一放在：

```
tests/
```


推荐结构：

```
tests/
├── unit/
├── integration/
└── api/
```


说明：

unit：

- 单元测试
- 测试独立模块


integration：

- 模块集成测试
- Agent 流程测试


api：

- FastAPI 接口测试


---

# 新功能测试要求

新增功能必须：

- 添加对应测试
- 保证已有测试通过
- 覆盖核心逻辑


禁止：

- 只修改代码不增加测试
- 删除已有测试绕过问题


---

# Backend 测试

FastAPI 接口必须测试：

- 正常请求
- 参数校验
- 异常处理
- 返回结果


要求：

- 使用测试客户端
- 不依赖真实外部服务


---

# Agent 测试

LangGraph Agent 测试重点：

- 状态流转正确
- 节点执行符合预期
- Tool 调用正确
- Memory 检索结果正确


测试时：

优先测试：

- Agent workflow
- Node 输入输出
- 状态变化


避免：

- 依赖真实 LLM 响应进行测试


---

# Memory 测试

Memory System 必须测试：

- 数据保存
- 数据检索
- 相似度查询
- 上下文构建


要求：

- 使用测试数据
- 保持测试可重复


---

# 数据库测试

数据库相关测试：

优先使用：

- 测试数据库
- 独立测试环境


禁止：

- 测试直接操作生产数据


---

# Mock 原则

外部依赖应该 Mock：

包括：

- LLM API
- 第三方服务
- 外部网络请求


测试重点：

- 业务逻辑
- 数据流转
- 状态变化


---

# 测试执行要求

代码提交前：

必须确保：

- 测试通过
- 无明显错误
- 核心流程可运行


运行：

```
pytest
```
